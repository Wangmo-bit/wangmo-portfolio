# 王墨的赛博分身 — PRD

> Mo Wang's personal portfolio — 让别人快速了解我、我的作品、我的社交平台。

## 1. 产品概述

### 1.1 产品定位

个人作品集网站，面向潜在合作者、用户、媒体，展示产品创造者 Mo Wang 的个人介绍与作品。

### 1.2 目标用户

- 对我的 App 感兴趣的潜在用户
- 想了解我背景的合作者 / 投资人
- 媒体记者（需要一页了解我的全部信息）

### 1.3 核心价值

- 一个链接了解全部：我是谁、做了什么、怎么联系
- 作品集中展示，带真实截图
- 媒体报道聚合，社交证明

### 1.4 商业模式

无商业化，纯个人展示站。

## 3. 视觉与体验风格

### 3.1 整体风格

- 赛博科技感，深色基底 + 霓虹色点缀
- 核心交互：切换作品时，左栏的背景色/氛围色跟随当前作品主题色过渡
- 克制的动效 — 颜色过渡、微光呼吸，不做 3D / 粒子等重特效
- 参考方向：[Framer Future 模板](https://www.framer.com/marketplace/templates/future-cyberpunk-portfolio-template/)的霓虹氛围 + 左右分栏的信息架构

### 3.2 布局

- **桌面端**：左右分栏 — 左侧固定（个人介绍，背景色随作品联动），右侧可滚动（作品列表）
- **移动端**：上下布局 — 个人介绍在上（精简），作品在下
- 响应式断点：桌面 ≥ 1024px，平板 768-1023px，手机 < 768px

### 3.3 作品主题色

| 作品 | 主题色 | 氛围 |
|------|--------|------|
| 赛博生活 / CyberLife | 青色 Cyan #00F0FF | 赛博朋克 |
| 爪印档案 | 暖橙 #FF8C42 | 温暖活力 |
| Lorebook | 紫罗兰 #8B5CF6 | 神秘知性 |

### 3.4 文案风格

- 简洁直白，不堆砌形容词
- 中文为主体语境，英文版保持同等质量（非机翻感）

## 4. 技术约束

### 4.1 技术栈

| 项目 | 选型 |
|------|------|
| 框架 | Astro |
| 样式 | Tailwind CSS |
| 语言 | TypeScript |
| 构建产物 | 纯静态 HTML/CSS/JS |
| 包管理 | pnpm |

### 4.2 部署

| 项目 | 方案 |
|------|------|
| 托管 | GitHub Pages |
| 域名 | wangmo.tech |
| HTTPS | GitHub Pages 自动证书 |
| CI/CD | GitHub Actions（推代码自动构建部署） |

### 4.3 依赖策略

- 零后端，纯静态站
- 内容通过 Markdown / JSON 文件管理，改内容不动代码
- 图片资源放在仓库内（作品截图压缩后体积可控）
- 三方依赖极简，只用 Astro + Tailwind，不引入 UI 组件库

## 6. v1.0 边界

### 做

- 个人介绍展示
- 三个作品展示（含截图 / 概念图）
- 社交平台链接（即刻、小红书、GitHub）
- 媒体报道板块（预留，有内容时显示）
- 中英双语切换
- 响应式布局（桌面 / 平板 / 手机）
- GitHub Pages 部署 + 自定义域名

### 不做

- 博客 / 原创文章系统
- 评论 / 留言功能
- 后端 / 数据库
- 用户登录
- 访问统计（v1.1 可加 umami 等轻量方案）
- 暗色模式切换（v1.1）

## 7. 边界情况与异常处理

| 场景 | 处理方式 |
|------|----------|
| 媒体报道为空 | 隐藏整个报道板块，不显示空状态 |
| 作品截图未准备好 | 显示占位图 + 作品文字描述 |
| 浏览器不支持现代 CSS | Astro 构建产物兼容主流浏览器，不额外兼容 IE |
| 图片加载慢 | 使用 Astro 内置图片优化，生成 WebP + 懒加载 |
| 域名未生效 | GitHub Pages 默认域名兜底（username.github.io） |
